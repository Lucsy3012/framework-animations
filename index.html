Sequences

animations.css provides you with a CSS only sequence function. This means that one universal trigger will call the inherited animation to all children. The delay of each animation can be altered with the --sequence-steps variable.

Setup

How to get animations.css running

Download the files

Luckily the library is very lightweight and can be downloaded here. The package contains the animations.min.css and an animate.min.js file. While the JS file is optional (but recommended) the CSS file is mandatory.

Then link them to your project:

                
                    <!-- in the <head> -->

                    <link rel="stylesheet" href="path/to/animations.min.css">
                
            
                
                    <!-- at the end of the page -->

                    <script src="path/to/animate.min.js"></script>
Use

How to use animations.css

1. Animate instantly

This option is best suited if you intend to animate a looping element rather than page revealing animations.

                
                    <!-- basic fade animation -->

                    <div class="animate"></div>

We can now add further options, for example:

                
                    <!-- looping rotateFull animation  -->

                    <div class="animate rotateFull loop"></div>

2. Animate when scrolled to

This option is a more dynamic and useful approach. It let's you start an animation when the user reaches the element. Be sure to include the animate.min.js when using.

                
                    <!-- basic fade animation -->

                    <div class="animate--js"></div>

We can now add further options, for example:

                
                    <!-- delayed cutIn animation -->

                    <div class="animate--js cutIn delay-s"></div>
Documentation

The nature of animations.css explained

Note that all of the following classes and values are automatically inherited by their ancestors. That means if animations get stacked on top of each other they obey the same ruleset, unless you overwrite properties you want to change for the child (and their ancestors). Here you can see how that could look like. Go ahead and inspect the linked example if you need further understanding.

                
                    <!-- this animation will "zoom in" with the "ease"-function -->

                    <div class="animate--js zoomIn ease">

                    

                        <!-- this animation will "rotate at the X axis" with the "ease"-function. But it will be delayed and take longer to complete than parent on top of that -->

                        <div class="animate--js rotateX delay-s duration-l"></div>

                    </div>

                    

                    <!-- together they make up for a whole new animation -->

Property Options Description

animation-timing-function --timing-function Overwrite to set a basic value for your animations.
.linear (Fallback) Changes the timing-function to the value "linear".
.ease Changes the timing-function to the value "ease".
.ease-in Changes the timing-function to the value "ease-in".
.ease-out Changes the timing-function to the value "ease-out".
.cubic-in Changes the timing-function to the value cubic-bezier(0.55, 0.06, 0.68, 0.19).
.cubic-out Changes the timing-function to the value cubic-bezier(0.22, 0.61, 0.36, 1).

animation-duration --duration Overwrite to set a basic value for your animations.
.duration-xs Changes the duration to the value 0.25s.
.duration-s Changes the duration to the value 0.5s.
.duration-m (Fallback) Changes the duration to the value 1s.
.duration-l Changes the duration to the value 2s.
.duration-xl Changes the duration to the value 5s.
.reset-duration Resets to the original value of --duration (default: unset) or the fallback (1s) if unset.
animation-delay --delay Overwrite to set a basic value for your animations.
.delay-xs Delays the start of the animation by 0.1s.
.delay-s Delays the start of the animation by 0.25s.
.delay-m Delays the start of the animation by 0.5s.
.delay-l Delays the start of the animation by 1s.
.delay-xl Delays the start of the animation by 2s.
.reset-delay Resets to the original value of --delay (default: unset) or the fallback (0s) if unset.
animation-direction --direction Overwrite to set a basic value for your animations.
.reverse Lets the animation play in reverse order.
.alternate Lets the animation play once forward and then reverse again.
Animation needs to be looping to have an effect.
.alternate-reverse Lets the animation play once reverse and then forward again.
Animation needs to be looping to have an effect.
.reset-direction Resets to the original value of --direction (default: unset) or the fallback (normal) if unset.
animation-fill-mode --fill-mode Overwrite to set a basic value for your animations.
.forwards Applies all CSS properties set within the animation after completion.
.backwards Applies all CSS properties set within the animation before the animation began.
.both Applies all CSS properties set within the animation before and after the animation played.
.reset-fill-mode Resets to the original value of --fill-mode (default: unset) or the fallback (both) if unset.
animation-iteration-count --iteration-count Overwrite to set a basic value for your animations.
.once Plays the animation once.
.infinite Plays the animation indefinitely, creating a looping animation.
.reset-iteration-count Resets to the original value of --iteration-count (default: unset) or the fallback (1) if unset.
animation-play-state --play-state Overwrite to set a basic value for your animations.
.paused Pauses the animation.
.running Starts or continues the animation.
.reset-play-state Resets to the original value of --play-state (default: unset) or the fallback (running) if unset.

Special classes

Name Class Description
Loop .loop Sets the respective properties to create a loop (forward only).
Pulsing loop .pulse Sets the respective properties to create a loop that pulses (forward then backwards again).
Sequence .sequence Add to .animate or .animate--js. If set it applies the animation to all direct children individually.
Each animation gets delayed by the value of --sequence-steps multiplied by the count at which the child sits.
.sequence-sliced Add to .sequence to split each character within the element into a separate <span>. Useful for title sequences.
--sequence-steps Defines the delay each children gets. Default: 0.1s
The first child has a delay of --sequence-steps * 0, the second --sequence-steps * 1 and so on.
Reset .reset Resets all animation properties to their original value or fallback.
